Securing And Sharing Files

Description

Normally every programmer deal with transferring of files. So, the problems they might face is hacking of data, a time delay due to transmission of data, copy of garbled data, etc. Our system provides a set of great utilities to the users. The security issues like Encryption, Decryption etc are provided, so that data is safely transferred without the fear of information leaking. Transmission of huge data files may engage network for a long time resulting the starvation of other clients. To avoid this inconsistency, the Split & Join utilities are provided in the system. The system provides various utilities:
 File Encryption/Decryption
 File Splitter/Merger
 File Comparison

Environment

Front-End: JSP , Bootstrap
Back-End: JAVA
Database: MySql
Server: Apache Tomcat 7.0

Algoritham

The idea of RSA is based on the fact that it is difficult to factorize a large integer. The public key consists of two numbers where one number is multiplication of two large prime numbers. And private key is also derived from the same two prime numbers. So if somebody can factorize the large number, the private key is compromised. Therefore encryption strength totally lies on the key size and if we double or triple the key size, the strength of encryption increases exponentially.
